# HIDEBAF In-Depth Cryptographic & Operational Documentation

## 1. Overview & Architecture

HIDEBAF (Highly Incubated Dragonian Encryptory Binary using an Archive of Files) is a high-security CLI-based file encryption utility implementing "scorched earth" multi-token redirection. Unlike conventional single-key encryption formats, HIDEBAF supports split key-material verification where a single `.HIDEBAF` archive can contain multiple authentication key slots (both real and decoy), mapping to different virtual sectors. 

```
                                  .HIDEBAF Archive Header
+------------------+------------------+---------------------+-------------------+
|  Magic Bytes     |  Version (u32)   |  Flags Field (u32)  |  Slot Count (u32) |
|  "HIDEBAF\0"     |  e.g., 0x00000001|  e.g., 0x00000000   |  e.g., 0x00000003 |
+------------------+------------------+---------------------+-------------------+
|  Slot 0 (212 B)  |  Slot 1 (212 B)  |  Slot 2 (Decoy)     |  Hint Length (u32)|
|  [Salt+Verifier] |  [Salt+Verifier] |  [Salt+Verifier]    |  e.g., 0x00000000 |
+------------------+------------------+---------------------+-------------------+
|  Payload Size    |  Encrypted Payload Sector (AES-256-GCM)                    |
|  (u64)           |  Contains compressed/uncompressed file entries             |
+------------------+------------------------------------------------------------+
```

## 2. Cryptographic Protocol Specification

### Key Derivation Function (KDF)
To translate a user-supplied token password into cryptographic keys, HIDEBAF employs `scrypt` key derivation:
- **Default Work Factor ($N$):** $16384$ (low latency desktop use).
- **Paranoid Mode Work Factor ($N$):** $65536$ (strengthened against GPU/ASIC brute-force attacks).
- **Block Size ($r$):** $8$
- **Parallelization ($p$):** $1$
- **Output Key Length:** $64$ bytes (split into a 32-byte Encryption Key and a 32-byte Authentication Verifier).

### Key Slot Architecture (Slot Size: 212 Bytes)
Each slot in the archive represents a potential door. The slot is composed of:
1. **Token Salt (16 bytes):** Random cryptographic salt generated using CSPRNG.
2. **Verifier (32 bytes):** Generated by hashing the derived scrypt key with SHA-256. This is matched during decryption to locate the appropriate key slot.
3. **Slot IV (12 bytes):** Initialization vector for GCM encryption of the sector reference metadata.
4. **Slot Tag (16 bytes):** Authenticated decryption tag of the GCM block.
5. **Slot Ciphertext (136 bytes):** Encrypted Sector Reference Metadata containing:
   - Sector Offset (8 bytes, Big-Endian uint64)
   - Sector Length (8 bytes, Big-Endian uint64)
   - Sector Key (32 bytes)
   - Salt/Padding (88 bytes of random padding)

## 3. Advanced Operational Protections

### 3.1 Anonymize Mode (Metadata Scrubber)
When **Anonymize Mode** is enabled (interactively toggleable in `settings` or under `secret_modes.anonymize` in `settings.yml`), the system wipes operational forensics:
1. **Filename Schematization:** Original filenames are replaced in the archive's internal manifest with sequential mappings:
   `file_0.ext`, `file_1.ext`, `file_2.ext`, etc.
2. **Path & Username Cleansing:** Directory structures, user-profile paths, and system-specific metadata are omitted from the bundle.
3. **Header Hint Suppression:** Any header `--hint` flag parameters are stripped to prevent plaintext leaks.
4. **Timestamp Sanitization:** Upon extraction (`reveal`), the modification and access timestamps of the resulting files are set to the Unix Epoch (`1970-01-01 00:00:00 UTC`), preventing forensic timeline analysis.
5. **Log Redaction:** Hooks are suppressed to keep operational telemetry out of log audits.

### 3.2 Self-Destruct Mode (Failsafe Payload Erasure)
Self-destruct protects local archives against offline brute-force attempts.
- **Fail Attempt Counter:** Tracks successive incorrect password attempts on a target archive.
- **Threshold Limit:** Customizable parameter `max_attempts` (defaults to 5).
- **Destruction Execution Rules:**
  - `always`: Triggers self-destruct immediately upon reaching the threshold limit on every attempt.
  - `firstTime`: Destructs on the first threshold breach, then writes a configuration flag to bypass subsequent wipes (preserving subsequent decoy files).
  - `custom` (number): Permits destruction a specific number of times per archive before permanently locking out self-destruct capabilities.
- **Shredding Payload:** Once self-destruct triggers, the archive binary payload is securely overwritten with an inline base64-encoded representation of a decoy image (the dragon logo, embedded directly in the compiled binary). The original content is completely destroyed.

## 4. Settings Configuration (`settings.yml`)

The settings file is located in the local AppData directory (`C:\Users\ROAYA\AppData\Local\HIDEBAF\Content\settings.yml`).

```yaml
# HIDEBAF Configuration Settings
theme: dragon

secret_modes:
  scorched_earth: false
  ghost: false
  paranoid: false
  anonymize: false

features:
  default_compress: true
  auto_redirection: false
  verbose_audit: true
  export_warnings: true

self_destruct:
  active: false
  max_attempts: 5
  mode: firstTime # Options: always, firstTime, or integer limits

disabled_plugins: []
disabled_commands: []
```

### Disabling Commands Interactively
Any core CLI command route (e.g., `reveal`, `export-tokens`, `verify`) can be programmatically disabled by adding its name to the `disabled_commands` block. Attempting to run a disabled command returns an immediate termination message:
```
✖ Command "reveal" has been disabled in HIDEBAF settings.
```
This configuration is toggleable interactively under the `settings` menu.

## 5. Command Reference & Lifecycle CLI Options

### `hide [options] <files...>`
Pack and encrypt target files.
- `--token <token>`: Explicitly define key-material bypass.
- `--no-compress`: Force-skip zlib payload compression.
- `--decoy <password>`: Inject decoy sectors pointing to garbage data blocks.
- `--export-tokens <path>`: Interactively write keys to a `.EBFATs` file.

### `reveal [options] <archive>`
Read, unlock key slots, decrypt GCM sector, and extract files.
- `-o, --output-dir <dir>`: Target location (will apply timestamp stripping if Anonymize Mode is on).

### `settings`
Toggles secret modes, configures the Self-Destruct trigger constraints, and lists core commands to block.
